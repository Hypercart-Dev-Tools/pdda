# Iron Triangle: Better

## Philosophy

Best-in-class experience from the first interaction. Smaller milestones, more
validation gates. The built-in forcing function is the **quality bar** — every
milestone must clear it.

## Milestone shape (governs MILESTONES.md)

- **Milestone count:** Keep the **full reference ladder (4–5 milestones)** and
  split further where a slice is large. Each milestone is a smaller vertical
  slice than a Faster build would ship.
  - **M1:** Core loop + one deliberate "delight moment." 2–3 weeks.
  - **M2:** Complete the loop + a polish pass. 2–3 weeks.
  - **M3:** Edge cases + performance + refinement. 2–3 weeks.
  - **M4:** Scale + advanced features.
- **Pacing:** 2–3 weeks per milestone — smaller scope, higher finish quality.
- **Validation-gate density:** **High — one gate per milestone** (usability,
  engagement, retention). Run a "would you pay for this?" test by M2.

## "Good enough" bar

- Every screen feels considered.
- Animations and transitions are intentional.
- Accessibility baseline met (keyboard nav, ARIA, focus management).
- Performance budget defined and tracked.
